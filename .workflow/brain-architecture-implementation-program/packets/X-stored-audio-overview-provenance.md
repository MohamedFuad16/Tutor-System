# Packet X: Stored Audio Overview Provenance

## Objective

Make stored chapter audio overviews visible in the same local artifact and citation-state ledger used for source cards, generated flashcards, and generated learning notes.

## Scope

- Add an `audio_overview` artifact kind.
- Create a pure helper that converts stored audio overview metadata into an `ArtifactRecord` plus linked `CitationState`.
- Seed built-in User Brain Architecture audio overview provenance when Admin loads.
- Update Admin copy, docs, book content, and focused artifact-record tests.

## Boundaries

- Do not call live TTS during playback.
- Do not claim the stored audio is verified evidence.
- Do not implement AWS/cloud synchronization.
- Keep generated Graphify artifacts regenerated by command only.
