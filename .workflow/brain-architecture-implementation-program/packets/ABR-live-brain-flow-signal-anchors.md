# Packet ABR: Live Brain-Flow Signal Anchors

## Objective

Turn the thirteen-signal Brain Flow Coverage cards from aggregate counters into
live proof cards. Admin should show compact request ids, row sources, latest
timestamps, and context PDF ids for each signal when the live ledger contains
evidence.

- Add compact signal-level evidence anchors to `BetaBrainFlowSignal`.
- Keep raw prompt context, transcript text, and private model internals out of
  the diagnostic payload.
- Render live request/source/PDF/timestamp chips on Admin Brain Flow cards.
- Keep missing signals honest with a no-anchor message.
- Update architecture/design copy to describe live signal anchors.

## Do Not

- Do not call live providers or paid APIs.
- Do not mark synthetic rehearsal rows as live evidence.
- Do not change the thirteen-signal readiness rules.
- Do not stage unrelated untracked workflow folders.
