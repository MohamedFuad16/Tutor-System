# Packet ABV: Coherent Live Proof Bundle

## Objective

Require the provider-key beta proof to show one coherent typed-chat request and
one coherent live-voice request, not only scattered green ledger rows.

## Scope

- Add a local coherent proof bundle builder for chat and voice request evidence.
- Keep provider calls out of scope.
- Surface the coherent bundle in Admin Beta Diagnostics.
- Strengthen focused tests so aggregate brain-flow readiness cannot hide
  request-scattered evidence.

- Workflow docs and regenerated Graphify artifacts.

## Acceptance

- A coherent proof bundle is ready only when one typed-chat request has context,
  retrieval, model, tool, evaluated mastery, transcript, and background-memory
  rows.
- The same is required for one live-voice request.
- The selected chat and voice requests must share more than one PDF context id.
- The selected chat and voice requests must share a saved local book/thread
  anchor.
- Provider-key proof includes the coherent bundle as a required live-ledger
  check.
- Tests prove aggregate 100% brain-flow signals can still fail coherent proof
  when rows are scattered across unrelated request ids.
