# P1 Interaction Purist

## Objective
Argue for the strongest practical Thinking Machines-style continuous tutor architecture LearningAI can implement without training a native interaction model.

## Ideology
The tutor should feel continuously present. Foreground and background layers should both be smart. The foreground layer handles live timing, interruption, and teaching flow; the asynchronous background layer handles retrieval, tool calls, generated artifacts, validation, and memory updates.

## Required Sources
- Thinking Machines interaction models article.
- Official OpenAI docs for Realtime, tools/function calling, background mode, agents/tracing if relevant.
- Any source explaining asynchronous/background task orchestration if needed.

## Expected Output
- Argument with citations.
- Architecture recommendation.
- Risks and constraints.
- Rebuttal points expected from other ideologies.
