# P5 Final Evaluation

## Decision
Accept the convergence: keep the two-layer strategy, but name it **continuous
tutor orchestration inspired by interaction models**.

## Accepted
- Fast foreground tutor plus asynchronous background workers.
- Retrieval, structured learner summaries, instructions, and evals before
  fine-tuning.
- BKT as the first auditable mastery baseline.
- DKT, AKT, LPKT, LKT, and newer LLM-based KT methods as research candidates.
- Tool calls with purpose, budget, timeout, trace, source state, and failure
  behavior.
- Privacy, tenant boundaries, consent, and audit logs as architecture.

## Rejected
- Native interaction-model training claims.
- Direct background mutation of mastery or learner truth.
- "Newest KT model is best" framing.
- Decorative diagrams with no explanatory job.
- Fine-tuning as the main adaptive-memory path.

## Integrated
- Added plain-English boundaries.
- Added source-of-truth table, citation states, contracts, captions, grouped
  source appendix, and stricter beta gates.
